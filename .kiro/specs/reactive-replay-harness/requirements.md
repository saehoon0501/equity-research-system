# Requirements Document

## Project Description (Input)

A **point-in-time counterfactual backtest engine** for the reactive CFD layer. See `.kiro/specs/reactive-replay-harness/brief.md` for the full discovery brief. Extracted from `walkforward-tuning-loop` at its `/kiro-validate-design` (operator decision 2026-05-30); data feasibility confirmed the same day by a `/deep-research` pass against the Massive API.

**Who has the problem.** `walkforward-tuning-loop` validates candidate trading configs by **in-process CPCV replay** (operator decision 2026-05-30, amends §14.6) — re-simulating each candidate over purged cross-validation partitions of realized history. A changed parameter makes a candidate take *different decisions* than the live system did, so the re-simulation must reconstruct each candidate's **own divergent decision-and-account path** — fetching point-in-time market data for instants/names the live system never traded. This re-simulation is the single largest, highest-risk net-new component, and every autonomous promotion's score rests on its fidelity; it was split out for boundary isolation, independent versioning/testing, and reuse (`in-session-monitor` and the eval-loop could also use it).

**Current situation.** The engine was specified inside `walkforward-tuning-loop`'s design (the `replay` "Subsystem" leaf) and is now extracted here. The cores it drives exist as specs — `reactive-signal-model` (decision core, DESIGNED), `survival-gate` (`admit`/`assess`, DESIGNED) — and `broker-cfd-adapter`'s `src/mcp/broker/paper.py` paper-fill sim is LANDED. Realized history + the champion's actual path come from `decision-trace-telemetry` (mig 048 landed) + `counterfactual_ledger`. **Data feasibility CONFIRMED (deep-research, Task wfho5d8um):** the Massive API (live Polygon.io rebrand, wire-compatible) serves every required historical type on the **Advanced/Business tier** — 1-second intraday bars (`/v2/aggs/ticker/{t}/range/...`), tick trades (`/v3/trades`), NBBO quotes (`/v3/quotes`), grouped daily (`/v2/aggs/grouped/...`), split-unadjusted bars (`adjusted=false`), delisted-name inclusion — back to 2003-09-10.

**What should change.** Build a deterministic, importable **leaf** backtest engine that, given a candidate config (a `ParamSnapshot` and/or `SurvivalParameters` delta, and/or a code version) + a historical window, reconstructs the candidate's counterfactual decision-and-account path by **driving the landed reactive + survival cores + the paper-fill sim** over **point-in-time** Massive historical inputs (no look-ahead; re-fetching where the candidate diverges from the champion; honoring §16.1 intraday-flat), and returns a structured per-period **outcome record** (decisions, fills at counterparty prices, P&L, survival events, predicted probabilities, realized labels). It must **credit cash dividends separately** for total-return P&L (Massive bars are split-unadjustable but never dividend-adjusted), and satisfy a **fidelity precondition**: replaying the champion's own version reproduces its realized ledger P&L within tolerance.

**In scope:** the counterfactual simulation (drive cores + paper-fill sim over a window for one config, reconstructing the divergent decision-and-account path); the point-in-time Massive historical-data access via a **direct REST client** (bars/trades/quotes/grouped-daily/dividends; split-unadjusted; delisted; no look-ahead; pagination) — NOT via the real-time `massive` MCP server; dividend crediting; the structured per-period outcome record; the champion-reproduction fidelity check.

**Out of boundary** (owned elsewhere — see brief): the CPCV partition scheme, the survival-net metric + calibration scoring, the DSR/PBO gate, the fit/trial-set, publish/audit (all `walkforward-tuning-loop`); live/real-time trading + the `massive` MCP path (daemon / `/micro`); the reactive/survival decision *logic* (imported pure cores, not owned); the trace/ledger schemas (`decision-trace-telemetry`, read-only).

**Constraints**: requires Massive **Advanced/Business** tier (procurement); **dividend-crediting** required for total-return P&L; **no look-ahead** point-in-time fetch + 50k-row pagination cap; **champion-reproduction fidelity** precondition; **P1/P14** leaf module, inner-ring testable in isolation with stub cores; DESIGNED-not-landed core deps (reactive/survival) → stub-tested + revalidate on their signatures; likely **no new migration** (read-only).

**Open items to resolve in requirements / before implementation** (the four deep-research live-probe items, unverified-not-absent): (1) SPY + named S&P 500 constituent symbol coverage (ETF coverage); (2) exact per-tier request rate limits + current pricing; (3) reference-endpoint (splits/dividends/tickers/market-holidays) entitlement per tier (200-vs-403); (4) delisted-ticker OHLC depth for very old names + "halted" status retrievability. Confirm with a one-shot live Massive API probe.
