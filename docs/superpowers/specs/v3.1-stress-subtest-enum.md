## Purpose

Lock the canonical list of stress sub-tests that pm-supervisor's §2.6 adversarial stress-test pass MAY fire, with explicit firing conditions citing specific upstream envelope field paths. Any sub-test emitted outside this enum triggers HARD FAIL via evaluator HG-29 (post-Phase B).

**The bug this closes:** Phase 5a Wave 2 A1-tight run `d9e4f537` emitted `kills_fired=1` with stated reason "§2.6 stress_failed on spot/IV divergence." That reason is NOT in the current §2.6 spec — the LLM invented a sub-test name. A7-tight (`8f0cc43c`) at near-identical DCF outputs emitted `kills_fired=0`, producing TRIM vs HOLD divergence on otherwise-identical conviction inputs. The asymmetry is structurally undetectable without an enum because the LLM can name a stress concept that doesn't exist.

## Method

The enum is **derived from the existing §2.6 spec text** (`.claude/agents/pm-supervisor.md` §2.6, lines 37-110), NOT proposed as new functionality. The current spec already enumerates the conditions under which `stress_passed/stress_open/stress_failed` should fire — Phase 0's job is to label each condition with a canonical ID + lock the cited upstream field path + lock the firing threshold + threshold direction.

Phase 0 does NOT add or remove stress sub-tests. It formalizes what is already implicit.

## Canonical stress sub-test enum (DRAFT — locked at Phase 0 attestation)

### STRESS_HELMER_POWER_ABSENT

**Source spec lines:** pm-supervisor.md §2.6 "Helmer-Power gate" — "If `len(strategic.helmer_powers_evidence) == 0` → record `stress_failed` with reason ..."

**Fires iff:**
- `quant_envelope.outside_view.corrected_divergence_pp > +2.0` (above-base-rate growth claim asserted), AND
- `len(strategic_envelope.helmer_powers_evidence) == 0`

**Cited upstream fields:**
- `quantitative-analyst.outside_view.corrected_divergence_pp` (field_type: percentage)
- `strategic-analyst.helmer_powers_evidence` (field_type: count via `len()`)

**Threshold + direction:**
- corrected_divergence_pp: `> 2.0`, direction "above"
- helmer_powers_evidence count: `== 0`, direction "equals"

**Routing:** `stress_failed` → feeds `kills_fired += 1`.

**Severity:** non-catastrophic by default; may escalate to catastrophic if conjunctively-triggered with reinvestment_moat quality_label D (see STRESS_REINVESTMENT_QUALITY_D_CONTRADICTION).

### STRESS_HELMER_POWER_UNDER_EVIDENCED

**Source spec lines:** pm-supervisor.md §2.6 "Helmer-Power gate" — "If Helmer Powers are claimed but `primary_source_citations` are missing, insufficient (<2), or fail the tier ≤ 2 quality check → record `stress_open` (NOT `stress_failed`) ..."

**Fires iff:**
- `quant_envelope.outside_view.corrected_divergence_pp > +2.0`, AND
- `len(strategic_envelope.helmer_powers_evidence) >= 1`, AND
- ANY of: `len(helmer_powers_evidence[i].primary_source_citations) < 2` OR `helmer_powers_evidence[i].primary_source_citations[*].source_quality_tier > 2`

**Cited upstream fields:**
- `quantitative-analyst.outside_view.corrected_divergence_pp` (percentage)
- `strategic-analyst.helmer_powers_evidence[*].primary_source_citations` (count + categorical)
- `strategic-analyst.helmer_powers_evidence[*].primary_source_citations[*].source_quality_tier` (count)

**Threshold + direction:**
- corrected_divergence_pp: `> 2.0`, "above"
- primary_source_citations count per evidence entry: `< 2`, "below"
- source_quality_tier per citation: `> 2`, "above"

**Routing:** `stress_open` → does NOT feed `kills_fired` (open ≠ failed).

### STRESS_REINVESTMENT_QUALITY_D_CONTRADICTION

**Source spec lines:** pm-supervisor.md §2.6 "Reinvestment-moat consumption" — "`quality_label: D` AND outside-view divergence > +2pp → **strong contradiction**: ... **Force `stress_failed` even if Helmer Powers are evidenced** ..."

**Fires iff:**
- `quant_envelope.reinvestment_moat.quality_label == "D"`, AND
- `quant_envelope.outside_view.corrected_divergence_pp > +2.0`

**Cited upstream fields:**
- `quantitative-analyst.reinvestment_moat.quality_label` (string_categorical)
- `quantitative-analyst.outside_view.corrected_divergence_pp` (percentage)

**Threshold + direction:**
- quality_label: `== "D"`, "equals"
- corrected_divergence_pp: `> 2.0`, "above"

**Routing:** `stress_failed` (catastrophic) → feeds `kills_fired += 1`. Overrides any STRESS_HELMER_POWER_ABSENT/UNDER_EVIDENCED outcome on the same claim ("the math wins this routing decision" per spec).

### STRESS_CAPITAL_LIGHT_CHAIN_BROKEN

**Source spec lines:** pm-supervisor.md §2.6 "Reinvestment-moat consumption" — "`quality_label: 'N/A capital-light'` AND `corrected_divergence_pp > +2pp` → ... force `stress_open` (not `stress_failed`) ..."

**Fires iff:**
- `quant_envelope.reinvestment_moat.quality_label == "N/A capital-light"`, AND
- `quant_envelope.outside_view.corrected_divergence_pp > +2.0`, AND
- `len(strategic_envelope.helmer_powers_evidence) >= 1` (otherwise STRESS_HELMER_POWER_ABSENT fires instead — disambiguation per /review-me iter 1 S3), AND
- NO entry in `strategic_envelope.helmer_powers_evidence[*].power_name` is in `{switching_costs, network_economies, branding}` (capital-light-relevant Powers)

**Cited upstream fields:**
- `quantitative-analyst.reinvestment_moat.quality_label` (string_categorical)
- `quantitative-analyst.outside_view.corrected_divergence_pp` (percentage)
- `strategic-analyst.helmer_powers_evidence[*].power_name` (string_categorical, set membership check)

**Threshold + direction:**
- quality_label: `== "N/A capital-light"`, "equals"
- corrected_divergence_pp: `> 2.0`, "above"
- power_name set intersection with capital-light set: `== empty`, "equals"

**Routing:** `stress_open` → does NOT feed `kills_fired`.

### STRESS_GENERIC_CLAIM_INVERSION_FAILED — residual category (S4 fix)

**Source spec lines:** pm-supervisor.md §2.6 step 1-3 (lines 41-49) — generic claim inversion procedure.

**Purpose:** preserve §2.6 step 1-3's authority to flag falsifiable claims OUTSIDE the 4 mechanical gates, while enforcing search-grounded evidence rather than freeform LLM judgment. Closes the "spot/IV divergence"-class loophole where the LLM invents a stress concept.

**Fires iff:**
- pm-supervisor identifies a load-bearing claim in cdd-lead/quant/strategic memo that is NOT covered by the 4 mechanical gates above, AND
- pm-supervisor performs a 1-line inversion + finds falsifying evidence is PRESENT in ANY of:
  - upstream envelopes (quant/strategic/catalyst-scout JSON)
  - `evidence_index` DB rows associated with the run_id
  - `analyst_briefs` DB rows for the ticker
  - **fresh external data pulls** via newly-granted pm-supervisor MCPs (per operator clarification 2026-05-20; see MCP grant expansion section below)
- Evidence MUST be cited via `searched_artifact_provenance` (see schema below)

**Cited evidence path:** variable (depends on which claim is inverted). MUST be one of:
- Existing-artifact resolution via restricted-path grammar: `quantitative-analyst.<path>` / `strategic-analyst.<path>` / `catalyst-scout.<path>` / `evidence_index.<row_uuid>.<field>` / `analyst_briefs.<brief_id>.content.<yaml_path>`
- Fresh-external-pull resolution: `mcp://<mcp_name>/<tool_name>?<query_params>` (canonical URI form) + cached response stored in `evidence_index` with `source_quality_tier <= 2`

**Threshold + direction:** must be specified per-claim, locked at emission time.

**Routing:** `stress_failed` → feeds `kills_fired += 1`. Severity is non-catastrophic UNLESS the inverted claim is moat-relevant or terminal-value-impairing (then `severity: catastrophic`, follows REINVESTMENT_QUALITY_D precedent).

**HARD requirement (closes Bug 3 loophole):**
- pm-supervisor MUST cite the artifact OR fresh-pull URI that contains the falsifying evidence
- pm-supervisor MUST NOT fire STRESS_GENERIC_CLAIM_INVERSION_FAILED based on "internal reasoning" alone — every emission requires a `searched_artifact_provenance` row in kills_fired_evidence[]
- Fresh external pulls MUST be cached in `evidence_index` BEFORE being cited (so downstream audit can verify the source)
- If no falsifying evidence is found after a reasonable search (existing artifacts + fresh pulls if invoked), the categorization is `stress_open` (not `stress_failed`); failure-by-LLM-fiat is rejected

**Rationale for the residual category:** §2.6 step 1-3's generic-claim-inversion authority is valuable for catching novel structural concerns (e.g., a new disclosure in a recent 8-K that wasn't picked up by the mechanical gates). Without this entry, that authority is eliminated. WITH this entry + search-grounding requirement (including fresh pulls), the authority is preserved AND extended; the freeform-LLM-stress-concept loophole is closed by requiring artifact provenance.

## Schema fields locked for kills_fired_evidence[] (Phase B implementation target)

Per consensus doc B.2:

```
kills_fired_evidence[]: array of objects, one per CITED FIELD per sub-test that fired:
  sub_test_name: <enum-value from this doc>
  severity: "catastrophic" | "non_catastrophic"     # per /review-me iter 1 S1
  upstream_envelope_uuid: <run_id of the cited upstream envelope OR null for fresh-pull>
  upstream_field_path: <restricted-grammar dotted path OR mcp:// URI for fresh-pull>
  field_type: "currency" | "percentage" | "ratio" | "count" | "string_categorical"
  threshold: <number | string>
  threshold_direction: "above" | "below" | "equals"
  observed_value: <number | string>
  searched_artifact_provenance: <object — for STRESS_GENERIC only; see schema below>
  narrative: <optional string, NEVER load-bearing>
```

**Severity defaults per enum entry (S1 fix):**
- STRESS_HELMER_POWER_ABSENT → `non_catastrophic`
- STRESS_HELMER_POWER_UNDER_EVIDENCED → `non_catastrophic` (routes stress_open, doesn't feed kills_fired)
- STRESS_REINVESTMENT_QUALITY_D_CONTRADICTION → `catastrophic`
- STRESS_CAPITAL_LIGHT_CHAIN_BROKEN → `non_catastrophic` (routes stress_open)
- STRESS_GENERIC_CLAIM_INVERSION_FAILED → `non_catastrophic` ALWAYS (S7 lock per /review-me iter 2; LLM-judged escalation removed. Rationale: the 4 mechanical gates handle catastrophic cases via REINVESTMENT_QUALITY_D. STRESS_GENERIC by definition catches non-enumerated claims; if a recurring catastrophic STRESS_GENERIC pattern emerges, that's a signal to add a new mechanical sub-test via /spec-approve, NOT to retain LLM-judged escalation that re-opens the Bug 3 surface)

Severity feeds the LOW conviction trigger per pm-supervisor.md §2.6 step 3.

**`searched_artifact_provenance` schema (STRESS_GENERIC only):**
```
{
  source_type: "envelope" | "evidence_index" | "analyst_briefs" | "fresh_external_pull",
  source_uri: <path or mcp:// URI>,
  retrieved_at: <ISO 8601 timestamp>,
  evidence_index_cache_uuid: <UUID, required if source_type=fresh_external_pull>,
  inverted_claim_text: <string — audit-trail ONLY, NEVER load-bearing for HG-29>,
  falsifier_text: <string — audit-trail ONLY, NEVER load-bearing for HG-29>
}
```

**Annotation (S9 lock per /review-me iter 2):** `inverted_claim_text` and `falsifier_text` are free-text fields for audit-trail color only. HG-29's deterministic validation operates on the tuple `(field_path, threshold, observed_value, threshold_direction)` — these free-text fields are NOT in that tuple and MUST NOT drive any gate decision. Adversarial post-hoc audit may read them for context; the gate ignores them.

**Per-field-emission convention (S5 rationale lock):**

For multi-field sub-tests (e.g., STRESS_HELMER_POWER_ABSENT cites BOTH `corrected_divergence_pp` AND `helmer_powers_evidence` count), emit ONE entry per cited field — i.e., the same sub_test_name can appear N times in the array if it cites N upstream fields. **Rationale:** HG-29 validates `(field_path, threshold, observed_value, threshold_direction)` as a deterministic tuple; packing N fields into one entry forces array-of-arrays validation which doesn't compose with the restricted-grammar field-path resolver. Per-field emission keeps each gate-check atomic and independently verifiable.

## Critical Architectural Findings (CAFs)

### CAF-A — Specialists-pull-data architectural protection inverted by STRESS_GENERIC fresh-pull capability (per operator clarification 2026-05-20)

**The inversion:** the prior architectural pattern was "specialists (quant/strategic/catalyst-scout) pull data; synthesizer (pm-supervisor) synthesizes already-produced artifacts." STRESS_GENERIC_CLAIM_INVERSION_FAILED with fresh-pull capability blurs this — pm-supervisor now has direct external-pull authority for stress-grounding.

**Operator rationale:** acknowledged at /grill-me Section 1 follow-up — generic claim inversion needs grounding flexibility, and limiting it to existing artifacts could miss novel disclosures (8-Ks filed after specialist runs, real-time price-action shifts, recent target-price revisions). Fresh pulls give pm-supervisor the same flexibility specialists have.

**Bug 3 surface-area implication:** pm-supervisor's LLM-non-determinism now extends to: (a) which claims to invert, (b) which sources to search, (c) which fresh pulls to invoke, (d) how to interpret pulled data. The mechanical-gate enum + content-level HG-29 still mechanically validates the 4 mechanical sub-tests, but STRESS_GENERIC retains LLM judgment room. This is an accepted trade-off per the operator's preservation-of-authority preference.

**Mitigation:** all fresh pulls must be cached in `evidence_index` with `source_quality_tier <= 2` BEFORE being cited; cached responses are subject to downstream contamination check (HG-1) and audit-trail verification. Audit-trail dimensions (b)/(c)/(d) become reviewable post-hoc.

**Resolved (S8 per /review-me iter 2 closure):** the inversion IS architecturally significant, not cosmetic. Actor identity affects (a) audit-trail attribution (which agent's run_id owns the pull), (b) prompt-caching boundaries (pm-supervisor's context now includes raw external data, expanding its attack surface for context-contamination), (c) HG-1 contamination check applies per-agent. Mitigation chain (evidence_index caching + HG-1 + audit-trail) is adequate per operator's accepted trade-off. Verdict: **significant but mitigated.** Future audit MUST verify cached fresh-pull provenance via evidence_index roundtrip; if HG-1 misses contamination from pm-supervisor's pulls, that's a Bug 8-class follow-up.
