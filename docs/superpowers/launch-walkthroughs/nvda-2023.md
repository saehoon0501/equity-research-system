## Expected Behavior per Architectural Lock

### Conviction rollup HIGH-gate (Phase 4 Q2 lock)

Per the post Phase 4 Q2 fix, conviction = HIGH requires ALL of:

1. Mode classifier `rule_clean = True` (no LLM tie-breaker dependency)
2. Debate consensus ≥ 4/5 ADD verdicts
3. Counterfactual retrieval archetype distribution: ≥2 SURVIVOR-leaning matches in top-3
4. Anchor-drift channels triggered: 0
5. Catalog HMAC integrity: PASS (no tampered rows in the active pool)

NVDA-2023 satisfies all 5 conditions. Conviction must roll up to HIGH.


### M-2 alert emission

A clean HIGH BUY produces an `unread_alerts` row with:
- `alert_type = 'recommendation_high_conviction'`
- `severity = 2`

This is operator-visible but not blocking. The operator must explicitly act
(via `/entry-check NVDA` and downstream sizing) before any execution.

### HMAC tamper-evidence (Section 7.2 invariant)

Same invariant as PLTR-2022: catalog rows feeding Layer 3 must be HMAC-verified
at load time. With NVDA-2023's SURVIVOR-shaped retrieval, a tampered row would
flip the verdict only by demoting a SURVIVOR top-3 case to NON-SURVIVOR — the
HMAC gate must catch that.
